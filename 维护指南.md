# LumenGuide 网站维护指南

> 基于 WinUI 设计语言的 Vue 3 内容展示平台。本指南教你如何添加、修改、删除文章，以及日常维护中常用的代码片段。

---

## 一、项目结构速览

```
LumenGuide/
├── src/
│   ├── App.vue                 # 应用入口：路由表、导航菜单、全局设置
│   ├── data/
│   │   └── pages.ts            # 站点通用页（首页/设置）+ 搜索/导航；从 chapters.ts 重新导出 chapter*
│   ├── pages/                  # 页面组件（含篇章文章）
│   │   ├── chapters.ts         # 卷/章定义：chapterPlan（卷名/图标/各卷章标题，支持单卷增删章）
│   │   ├── VolumeShell.vue     # 各卷共用壳：WinPage 包裹 + 锚点定位 + 章排版样式
│   │   ├── Volume1.vue ~ Volume9.vue  # 9 卷页面组件，每卷用 Vue 模板写各章正文
│   │   ├── HomePage.vue        # 首页（含项目介绍、九大篇章、最近访问、我的收藏）
│   │   └── SettingsPage.vue    # 设置页
│   ├── components/             # WinUI 组件库
│   │   ├── WinNavigationView.vue # 导航组件
│   │   ├── WinSearchBox.vue    # 搜索框
│   │   ├── WinPage.vue         # 页面容器（标题栏、收藏、主题切换）
│   │   └── ...                 # 其他 WinUI 控件
│   ├── composables/
│   │   └── useFavorites.ts     # 收藏逻辑
│   ├── styles/
│   │   ├── theme.css           # 主题变量（颜色、字体、背景）
│   │   └── animations.css      # 页面切换动画
│   └── main.ts                 # 入口文件
├── dist/                       # 构建产物
├── index.html                  # HTML 模板
└── vite.config.ts              # Vite 配置
```

维护文章时，你主要改三个地方：
1. `src/pages/` — 新增/修改/删除页面文件
2. `src/data/pages.ts` — 登记页面元数据（否则搜索找不到）
3. `src/App.vue` — 路由表 + 导航菜单

## 二、添加一篇新文章

### 步骤 1：复制模板创建文件

复制 `src/pages/ArticleTemplate1.vue`，改名为 `src/pages/MyArticle.vue`。

文件基本结构：

```vue
<template>
  <WinPage title="文章标题" description="文章副标题" pageKey="myarticle">
    <div class="article-content">
      <h2 class="section-title">一、引言</h2>
      <p>这里写正文内容。</p>

      <h2 class="section-title">二、核心内容</h2>
      <p>继续写内容。</p>

      <div class="article-meta-bottom">
        <span class="meta-item"><span class="icon">&#xE77B;</span> 作者名</span>
        <span class="meta-item"><span class="icon">&#xE823;</span> 2026-07-06</span>
      </div>
    </div>
  </WinPage>
</template>

<script setup>
import WinPage from '../components/WinPage.vue';
</script>

<style scoped>
/* 可直接复制 ArticleTemplate1.vue 的样式 */
.article-content {
  font-size: 16px;
  color: var(--text-primary);
  line-height: 1.8;
}
.section-title {
  font-size: 22px;
  font-weight: 600;
  margin: 32px 0 16px 0;
  color: var(--text-primary);
  padding-bottom: 8px;
  border-bottom: 1px solid var(--stroke-divider);
}
</style>
```

**关键字段：**
- `title`：页面顶部大标题
- `description`：标题下方的灰色副标题
- `pageKey`：唯一标识，用于收藏和搜索。只能是英文/数字，例如 `myarticle`。

### 步骤 2：在路由表中注册

打开 `src/App.vue`，在 `pageMap` 里加入新页面：

```js
import MyArticle from './pages/MyArticle.vue';

const pageMap = {
  home: HomePage,
  myarticle: MyArticle,   // ← 新增
  settings: SettingsPage,
};
// 篇章测试页由 pages.ts 的 chapterTestPages 自动批量注册，无需手写每一条
```

### 步骤 3：登记页面元数据

打开 `src/data/pages.ts`，在 `allPages` 数组里添加：

```ts
export const allPages: PageMeta[] = [
  // ... 已有页面
  {
    key: 'myarticle',
    icon: '\uE8A5',               // Segoe Fluent Icons 图标码
    title: 'My Article',
    titleZh: '我的文章',
    desc: 'A custom article page.',
    descZh: '一篇自定义文章。',
    keywords: ['article', 'custom', '文章', '自定义']
  },
];
```

`icon` 用的是 Segoe Fluent Icons 的 Unicode 码。常用图标：
- `\uE80F` 首页
- `\uE8A5` 文档/文章
- `\uE946` 笔记
- `\uE713` 设置
- `\uE735` 收藏（实心星）
- `\uE734` 未收藏（空心星）

### 步骤 4：加入导航菜单

`src/App.vue` 的 `navMenuItems` 是左侧导航菜单。新页面可作为一级项加入：

```js
const navMenuItems = [
  { value: 'home', icon: '\uE80F', label: '首页' },
  ...chapterNavItems,                  // 九大篇章分组（来自 pages.ts，自动生成）
  { value: 'myarticle', icon: '\uE8A5', label: '我的文章' },  // ← 新增独立页面
];
```

> 注意：当前项目已**移除「技术文章」分组**。篇章（九大篇章）通过 `chapterNames` 自动生成导航，不要手动在 `navMenuItems` 里写死。

### 步骤 5：检查搜索分组

`src/data/pages.ts` 的 `pageGroups` 控制搜索结果如何分类。当前 `sitepages` 分组是写死的，新页面要加进去：

```ts
export const pageGroups: PageGroup[] = [
  {
    key: 'sitepages', icon: '\uE80F', label: '网站页面', children: [
      'home', 'myarticle'   // ← 新增
    ]
  },
  // chapters 分组会自动包含全部 18 个篇章测试页，无需手写
];
```

### 步骤 6：验证

```bash
# 构建检查
node "./node_modules/vite/bin/vite.js" build

# 本地预览
node "./node_modules/vite/bin/vite.js" --host
```

## 三、修改文章

直接编辑 `src/pages/ArticleTemplate1.vue`（或对应文章文件），保存后开发服务器会自动刷新。

可修改的内容：
- `<WinPage>` 的 `title`、`description`
- 正文 HTML（标题、段落、列表、代码块等）
- `<style scoped>` 中的样式

常用组件：
- `<WinExpander header="标题">`：可折叠面板
- `<WinButton>`：按钮
- `<WinImage src="..." />`：图片
- `<WinHyperlinkButton>`：链接按钮

## 四、删除文章

反向执行“添加文章”的步骤：

1. 删除 `src/pages/MyArticle.vue`
2. 在 `src/App.vue` 的 `pageMap` 中删除对应行
3. 在 `src/App.vue` 的 `navMenuItems` 中删除对应子项
4. 在 `src/data/pages.ts` 的 `allPages` 中删除对应对象
5. 在 `src/data/pages.ts` 的 `pageGroups` 中删除对应 key

## 七、常见问题

**Q：新文章搜索不到？**
A：检查三点：`allPages` 数组里是否登记、`pageGroups` 分组里是否加入 key、`App.vue` 的 `pageMap` 是否导入。

**Q：收藏按钮不显示？**
A：`WinPage` 必须传 `pageKey`。

**Q：导航在小屏下显示异常？**
A：确认没有旧版 `winui-force-small` 残留在 localStorage。按 F12 → Application → Local Storage 查看并删除。

**Q：改完后页面没变化？**
A：Vite 开发服务器有时热更新失败，按 **Ctrl + F5** 硬刷新。

## 八、篇章（文章）维护

首页的「九大篇章」网格和左侧导航里的 9 个卷分组，背后是**数据驱动**生成的。每个卷（volume）对应若干章（chapter），每章是一篇文章（`ch{n}-{t}`），导航点某章会进入该卷的 `Volume{n}.vue` 页面并锚点定位到对应章（共用 `VolumeShell.vue` 壳）。

### 1. 文件职责

| 文件 | 作用 |
|------|------|
| `src/pages/chapters.ts` → `chapterPlan` | 9 卷的定义（卷名 / 图标 / 各卷章标题）。**增删卷、增删某卷的章都只改这里** |
| `src/pages/Volume{n}.vue` | 第 n 卷的页面组件，用 Vue 模板写该卷各章正文；每章一个 `<section id="ch{n}-{t}">` |
| `src/pages/VolumeShell.vue` | 各卷共用的壳：`<WinPage>` 包裹 + 挂载后按 `pageKey` 锚点滚动到对应章 + 集中章排版样式 |
| `src/data/pages.ts` | 网站通用页（首页 / 设置）+ 搜索 / 导航逻辑；从 `chapters.ts` 重新导出 `chapterPlan / chapterNames / chapterArticles / chapterGroups` |
| `src/App.vue` → `pageMap` | 各章 key 映射到所属卷的 `Volume{n}` 组件（`chapterArticles.forEach(p => pageMap[p.key] = volumeComponents[n])`），导航点章即进卷页并定位 |

### 2. 改一个卷的名字 / 图标

打开 `src/pages/chapters.ts`，改 `chapterPlan` 对应那一行的 `name` / `icon` 即可，导航和首页同步更新：

```ts
export const chapterPlan: VolumeDef[] = [
  { name: '第一卷 · 开箱与初见', icon: '\uE7B8', chapters: ['第1章', '第2章', '第3章'] },
  // ……改这里（卷名 / 图标 / 各卷章标题）
];
```

### 3. 增加 / 删除一整个卷（篇章）

- **增加一卷**：
  1. `src/pages/chapters.ts` 的 `chapterPlan` 数组加一行，例如
     `{ name: '第十卷 · 理解数字世界', icon: '\uE8CC', chapters: ['第1章', '第2章', '第3章'] }`
     它自动成为第 10 卷，编号 n=10。
  2. 新建 `src/pages/Volume10.vue`（复制 `Volume1.vue` 结构，把 `<section id="ch10-1">` 等改好），在其中用 Vue 模板写正文；不建则打开该卷会白屏（无组件可渲染）。
  3. 导航分组、首页九宫格、搜索结果会自动多出这一卷的对应章数，无需改其它地方。
- **删除一卷**：反向操作——从 `chapterPlan` 删那一行 + 删 `src/pages/Volume{n}.vue`。其余自动消失。

### 3.1 增删「某一卷里的某一章」（单章）

当前模型**每卷自带 `chapters` 数组**，所以单卷单独增删章完全支持：
- **给某卷加一章**：在该卷 `chapters` 数组追加一项（如 `['第1章', '第2章', '第4章', '第5章']`），并在 `src/pages/Volume{n}.vue` 里补一个 `<section id="ch{n}-4">`、`<section id="ch{n}-5">` 区块写正文。导航 / 首页 / 搜索自动跟随。
- **删某卷一章**：从 `chapters` 数组移除该项，并删 `Volume{n}.vue` 里对应的 `<section id="ch{n}-{t}">` 区块（不删区块也只是不显示，无报错）。

> 注意：章 key 始终按 `ch{卷号}-{章序号}` 从 1 连续编号，**章序号以数组下标 +1 计算**，标题文本（如「第4章」）可自定义，与数组位置绑定。

### 4. 给某一章单独换图标 / 描述

`chapterArticles` 是在 `chapterPlan` 循环里生成的，每章图标取自所属卷的 `icon`。想给某一章单独换图标或描述，在 `src/pages/chapters.ts` 生成循环结束后覆盖即可：

```ts
const p = chapterArticles.find(x => x.key === 'ch3-2');
if (p) p.icon = '\uE770';   // 换成「设备」图标
```

### 5. 给篇章文章填真实内容（用 Vue 组件写正文）

**正文就是用 Vue 模板写**，放在 **`src/pages/Volume{n}.vue`** 里，每卷一个组件，最好识别。每个组件结构如下（复制 `Volume1.vue` 即可）：

```vue
<template>
  <VolumeShell>
    <section class="chapter" id="ch1-1">     <!-- id 必须 = 章 key，用于锚点定位 -->
      <h2 class="chapter-title">第1章 · 开箱初体验</h2>
      <div class="chapter-body">
        <p>正文段落……</p>
        <h3>小标题</h3>
        <ul><li>列表项一</li><li>列表项二</li></ul>
        <p>支持 <b>加粗</b>、<i>斜体</i>、<a href="#">链接</a>、<code>代码</code> 等常见标签。</p>
      </div>
    </section>
    <!-- 更多章继续加 <section id="ch1-2"> …… -->
  </VolumeShell>
</template>

<script setup lang="ts">
import VolumeShell from './VolumeShell.vue';
</script>
```

要点：
- `<section>` 的 `id` 必须是章 key（`ch{n}-{t}`），导航点章时 `VolumeShell` 会自动 `scrollIntoView` 到它。
- 章节标题写在 `.chapter-title`（h2），正文包在 `.chapter-body` 里；排版样式（字号 / 行高 / 列表 / 引用 / 代码 / 表格）由 `VolumeShell.vue` 的 `:deep()` 统一提供，**改一次全局生效**。
- 想加哪一章的内容，就到对应 `Volume{n}.vue` 里找到那个 `<section id="ch{n}-{t}">` 往 `.chapter-body` 写即可，无需改 `chapters.ts` 或其它地方。

> 已内置示例：第一卷 `src/pages/Volume1.vue` 的 `ch1-1` 已填了一段示例正文，打开即可看到效果。

**方式 B（独立页面，完全自定义版式）**：若某篇需要跟模板完全不同的排版，可单独建组件 `src/pages/Chapter3Page.vue`，并在 `App.vue` 的 `pageMap` 里注册 `ch3-1: Chapter3Page`（覆盖默认的 `Volume3`）。

### 6. 收藏功能说明

- 篇章文章**默认就有收藏按钮**：`VolumeShell.vue` 用 `<WinPage :pageKey="pageKey">` 包裹（pageKey 即当前章 key `ch1-1`…`ch9-3`），右上角自动出现爱心，点某章即收藏该章。
- 收藏状态走全局 `useFavorites`，点过的文章会显示在首页「我的收藏」和搜索面板的收藏区。
- 其他想要收藏的页面，也用 `<WinPage pageKey="某key">` 包裹即可（参考 `SettingsPage`）。

### 7. 让某篇文章不出现在「最近访问」

访问任意页都会进首页的「最近访问」（`App.vue` 的 `watch(currentPage)`）。若不希望篇章文章污染该列表，在 `App.vue` 的 watch 里加一行排除：

```js
watch(currentPage, (newPage) => {
  if (!newPage || newPage === 'home') return;
  if (newPage.startsWith('ch')) return;   // ← 排除所有篇章文章
  // ……原有写入逻辑
});
```
