## 第四轮修复（10:54）

### 搜索弹窗模糊效果修复
- **根本原因**：`--flyout-bg` 值为 `rgba(252,252,252)` / `rgba(44,44,44)`，完全不透明（无 alpha 通道），导致 `backdrop-filter: blur(20px)` 虽然设置了但被不透明背景完全遮挡，视觉上看不到模糊
- **修复**：theme.css 中所有 `--flyout-bg` 改为半透明
  - `@property` initial-value: `rgba(252,252,252,1)` → `rgba(252,252,252,0.8)`
  - 浅色主题: `rgba(252,252,252)` → `rgba(252,252,252,0.8)`
  - 深色主题: `rgba(44,44,44)` → `rgba(44,44,44,0.8)`
- **影响范围**：所有使用 `--flyout-bg` 的组件（WinSearchBox、WinFlyout、WinContentDialog、WinComboBox 等）现在都能正确显示亚克力模糊效果
- 这是原项目的 bug：设置了 backdrop-filter 但背景不透明

## 第二十五轮（19:21）：实现「单卷单独加/删一章」(chapterPlan 模型)

### 需求
用户："支持只想给某一卷单独加/删一章一下" —— 要能只改某一卷的章数，而不影响其他卷。旧模型（全局 `CHAPTERS_PER_VOLUME=3`）不支持，必须先改造。

### 做法
- `src/data/pages.ts` 重构为**数据驱动 + 每卷自声明章**：
  1. 新增接口 `VolumeDef { name; icon; chapters: string[] }`，`chapters` 长度即该卷章数（支持单卷不同章数）。
  2. 新增 `chapterPlan: VolumeDef[]`（9 卷，每卷 `chapters: ['第1章','第2章','第3章']`）。
  3. 删除 `CHAPTERS_PER_VOLUME` 常量与 `chapterIcons` 数组；`chapterNames` 改为 `chapterPlan.map(v => v.name)` 导出（兼容旧引用）。
  4. `chapterArticles` 生成循环改为遍历 `chapterPlan`，每卷用 `vol.chapters.forEach((chTitle, t) => key = ch${i+1}-${t+1})` 生成；`icon` 取该卷 `vol.icon`（每卷主题图标）。
  5. `chapterGroups`（搜索分组 / 导航分组）改为 `chapterPlan.map`，`children = vol.chapters.map((_, t) => ch${i+1}-${t+1})`。
- grep 确认无 `CHAPTERS_PER_VOLUME` / `chapterIcons` 残留；正文文件仍是 `src/content/volume{n}.ts`（glob `../content/volume*.ts` 不变）。

### 验证
- `vite build` ✅ 零错误（53 modules，CSS 60.45KB / JS 143.43KB）。

### 用法（已对用户说明）
- **给某卷加一章**：在 `chapterPlan` 对应卷的 `chapters` 数组追加一项（如 `['第1章','第2章','第4章','第5章']`），并在 `volume{n}.ts` 补 `'ch{n}-4'`、`'ch{n}-5'` 正文 key。导航/首页/搜索自动跟随。
- **删某卷一章**：从 `chapters` 数组移除该项，并删 `volume{n}.ts` 里对应 key（不删正文 key 也只是不显示，无报错）。
- 注意：章 key 始终按 `ch{卷号}-{章序号}` 从 1 连续编号，**章序号以数组下标 +1 计算**，标题文本（如「第4章」）可自定义，与数组位置绑定。

## 第二十八轮（19:50）：修刷新回首页 + 每章独立页面

### 需求
用户："为什么在文章里面刷新以后就到主页面了，每一章都要在不同的页面！" → 两个诉求：(1) 刷新文章会跳回首页；(2) 每一章应是独立页面。

### 做法
- `src/App.vue`（纯 JS）：`currentPage` 初始化自 `localStorage.getItem('winui-current-page') || 'home'`（加了 `typeof localStorage` 保护），并新增 `watch(currentPage)` 写回 `localStorage`。刷新后停留在当前章，不再回首页。（注意 App.vue 仍是 `<script setup>` 无 lang=ts，勿加类型注解。）
- `src/pages/VolumeShell.vue`：
  - 导入 `chapterPlan`，按 `chapterPlan.flatMap` 生成 `orderedKeys`（全章顺序），计算 `prevKey / nextKey`，`goTo(k)` 设 `currentPage.value = k`。
  - `scrollToCurrent` 改为：先把 `.volume` 下所有 `.chapter` 用 `el.style.display` 隐藏（非当前章 `display:none`），只显示 `pageKey` 对应的章 → **每章成为独立页面**；再 `scrollIntoView` 到当前章。
  - 模板 `<slot/>` 后加「上一章 / 下一章」按钮（`.chapter-nav` / `.chapter-nav-btn` 样式）。
- 这样：导航点某章 = 独立页；章内底部可翻上/下一章；刷新停在当前章。

### 验证
- `vite build` ✅ 零错误（64 modules）；`vue-tsc` 通过；dev server 模块 transform 200 无错。

### 结论
- 刷新保持当前文章；每章现在是独立聚焦页面（其余章隐藏），并配上下章导航。若用户其实想保留「一卷内看全部章的长页面」，去掉 VolumeShell 里的 `display:none` 隐藏逻辑即可还原。
